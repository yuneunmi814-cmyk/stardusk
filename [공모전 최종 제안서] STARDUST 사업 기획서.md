## 1. 서비스 요약

**STARDUST**는 사용자가 어디에 머물든, 그곳에서 올려다본 하늘을 짧은 영상으로 담으면
**① 좌표를 관광 명소로 자동 매핑**하고 **② 그 순간의 하늘빛을 K-Means 알고리즘으로 분석해
'#5794E4 · 맑은 오후' 같은 색상/감정 라벨을 자동으로 태깅**한 뒤, 같은 시간대·같은 하늘 아래
머문 사람들을 하나의 별자리처럼 연결하는 **감성 라이프 로그 서비스**다.

기존 관광 앱이 "어디를 갈까"(목적지 탐색)에 집중했다면, STARDUST는 **"지금 내가 머문 이 자리"**를
별로 승화시킨다. 새벽·푸른 낮·노을·고요한 밤 — 하루 24시간 모든 시간대의 하늘이 콘텐츠가 되며,
한국관광공사 국문/영문 OpenAPI를 융합해 **내국인과 외국인 관광객 모두에게** 같은 명소를
각자의 언어로 보여준다.

### 1.1 핵심 사용자 여정 (단 3초)

```
[셔터 ON] ──▶ [하늘 영상 촬영] ──▶ 끝.
                    │
                    ▼ (이후 전부 자동 — 유저는 아무것도 입력하지 않음)
   ┌──────────────────────────────────────────────────────┐
   │ • GPS 좌표 → PostGIS 최근접 관광 명소 자동 매핑          │
   │ • 영상 첫 프레임 상단(하늘) → K-Means 대표색 추출        │
   │ • HSV 분석 → '#5794E4 / 맑은 오후' 감정 라벨 자동 부여   │
   │ • 내 프로필 아바타 별빛(Glow)이 그 하늘색으로 자동 번짐   │
   │ • 영상 EXIF 위치 메타데이터 제거 후 안전하게 업로드        │
   └──────────────────────────────────────────────────────┘
```

## 2. 핵심 기능 및 UX — '선택 마찰을 최소화한 하이브리드 탐색'

### 2.1 문제 정의: 탐색은 너무 복잡하거나, 너무 막연하다

관광 앱의 탐색 UX는 양극단에서 실패한다. 한쪽은 **복잡한 입력의 늪**이다 — 지역을
드롭다운으로 고르고, 카테고리 탭을 넘기고, 키워드를 치고, 정렬을 바꾸는 단계마다
사용자가 이탈한다. 반대쪽은 **막연한 자동화**다 — "알아서 추천"만 하면 정작 *내가 원하는
그곳*을 찾을 길이 없어 답답하다. STARDUST의 해법은 **배달앱식 하이브리드 탐색**이다:
처음엔 GPS로 마찰 없이 시작하되, 원하면 **지도·리스트·검색·지역 필터**로 즉시 파고든다.
**'입력 Zero'가 아니라 '선택 마찰 최소화' — 자동으로 시작하고, 손쉽게 좁힌다.**

### 2.2 ① 위치 설정 뷰 — GPS 자동 지정, 그러나 언제든 내가 바꾼다

앱 구동 시 현재 GPS 위치로 **탐색 기준점이 자동 지정**된다. 하지만 여행지를 미리
둘러보거나 다른 동네를 탐색하고 싶을 때, 사용자는 배달앱처럼 직관적으로 위치를 바꾼다.

```
 ┌────────────────────────────────────────┐
 │  🔍 도로명·건물명 검색 (MKLocalSearch)    │   ← 직접 검색
 ├────────────────────────────────────────┤
 │                                        │
 │              📍 (중앙 고정 핀)            │   ← 지도를 움직이면
 │            지도를 드래그해 이동           │      핀 아래 좌표가 갱신
 │                                        │
 ├────────────────────────────────────────┤
 │  📌 강릉시 경포동                         │   ← 역지오코딩 위치명
 │  [        해당 위치로 시작하기        ]    │   ← 명시적 확정
 └────────────────────────────────────────┘
```

- **자동 시작**: `LocationProvider`(CoreLocation)가 권한 요청 후 1회 좌표를 잡아 카메라를 이동.
- **직접 검색**: `MKLocalSearch`로 도로명·건물명을 입력하면 후보 리스트 → 탭 시 핀 이동.
- **지도 핀 이동**: iOS 17 `onMapCameraChange`로 중앙 고정 핀 아래 좌표를 실시간 갱신하고
  `CLGeocoder` 역지오코딩으로 "강릉시 경포동" 같은 위치명을 표시.
- **명시적 확정**: **"해당 위치로 시작하기"** 버튼으로만 기준점이 확정(`AppLocation.confirm()`)되어,
  의도치 않은 위치로 탐색이 시작되는 일이 없다.

### 2.3 ② 상단 듀얼 탭 — 지도로 탐색 / 리스트로 탐색

확정된 기준 위치를 중심으로, 사용자는 상단 세그먼트 탭에서 **두 가지 탐색 방식**을 자유롭게 오간다.

```
   [ 지도로 탐색 ]  |  [ 리스트로 탐색 ]      ← 상단 듀얼 탭(Segmented)
 ┌──────────────────┐   ┌──────────────────┐
 │   ★    ★         │   │ 🔍 통합 검색창      │
 │      ★    ★ ←터치 │   │ [시/도▾][시/군/구▾] │  ← 지역 필터
 │  ★      ┌───────┐│   ├──────────────────┤
 │         │명소카드 ││   │ 🖼 경포해변  1.2km  │
 │         │길안내 ▶││   │ 🖼 오죽헌    2.4km  │
 │         └───────┘│   │ 🖼 정동진    8.0km  │
 └──────────────────┘   └──────────────────┘
```

#### 지도로 탐색 — 별이 뜬 지도에서 손끝으로 발견

- 기준 위치 주변 관광 명소가 **별 마커**(`★`)로 지도 위에 표출된다(`/tour/spots` · PostGIS `ST_DWithin`).
- 마커를 터치하면 하단에 **OpenAPI 기반 장소 카드**가 슬라이드 업 — 대표 이미지(`firstimage`),
  명소명(`title`), 주소(`addr1`), 거리를 한눈에.
- 카드의 **"여기로 길안내"**를 누르면 §3.5의 국산 지도 핸드오프(네이버·카카오·티맵 딥링크)로 연결되고,
  **`[자세히 보기 🎧]`**(§2.5)로 오디오 도슨트를 바로 들을 수 있다.

#### 리스트로 탐색 — 읽기 좋게 정렬, 필터·검색으로 빠르게 좁힌다

- 주변 명소를 **썸네일 + 명소명 + 주소 + 거리**의 가독성 높은 리스트로 정렬(거리순).
- 상단 **[지역별 선택 필터]**: `시/도`·`시/군/구` 2단 드롭다운(`/tour/regions`로 동적 구성)으로
  원하는 권역만 즉시 추린다.
- 상단 **[관광지 통합 검색창]**: 명소명·주소를 직접 타이핑하면 `ILIKE` 기반 통합 검색(`/tour/search`)으로
  원하는 목적지를 빠르게 뚫어본다.

> 💡 **심사 포인트:** 자동(GPS) → 시각(지도) → 정밀(리스트·필터·검색)의 **3단 깊이**를
> 하나의 일관된 화면 흐름에 담았다. 막연하지 않으면서도 복잡하지 않은, **대중적인 탐색 UX**다.

### 2.4 자동으로 남는 것은 그대로 — 묻지 않아도 채워지는 메타데이터

탐색은 사용자가 주도하되, **기록 단계의 마찰은 여전히 0**이다. 하늘을 담는 순간 아래는 전부 자동이다.

| 사용자가 **안 하는** 것 | 대신 시스템이 **자동으로** 하는 것 | 근거 기술 |
|---|---|---|
| 제목/설명 입력 | 시간대·명소 기반 맥락 자동 구성 | 좌표→명소 매핑 |
| 위치명 검색·선택 | GPS 좌표 → 최근접 관광 명소 자동 매핑 | PostGIS `ST_DWithin` / KTO OpenAPI |
| 감정·분위기 태그 선택 | 하늘빛 분석 → '맑은 오후/따뜻한 노을' 자동 태깅 | K-Means + HSV 분류 |
| 프로필 색상 커스터마이즈 | 올려다본 하늘색이 아바타 별빛으로 자동 번짐 | `sky_color_hex` → SwiftUI Glow |
| 위치 메타데이터 노출 걱정 | 업로드 영상 EXIF 위치 정보 자동 제거 | 서버 재인코딩 EXIF strip |
| 영상 재생 버튼 누르기 | 화면 중앙 셀 자동 무음 재생 / 벗어나면 자동 해제 | 뷰포트 AVPlayer 최적화 |

**유저는 원하는 곳을 손쉽게 찾고, 하늘을 향해 셔터를 누른다. 나머지는 STARDUST가 알아서 별로 만든다.**

### 2.5 체류 시간을 늘리는 두 개의 디테일 — '강요 않는 알림'과 '듣는 안내'

탐색·기록의 마찰을 줄이는 것만으로는 부족하다. **재방문과 체류를 만드는 건 디테일**이다.
STARDUST는 사용자를 떠나게 만드는 두 지점 — 알림 피로와 정보 과부하 — 을 정면으로 끊는다.

**① 도착 팝업의 단순화·제어 — '알림 공해'를 원천 차단**

지오펜스(`CLCircularRegion`, 반경 40m)로 목적지 도착을 감지하면, 군더더기 없이 **딱 3개의 버튼**만 띄운다.

| 버튼 | 동작 |
|---|---|
| `[촬영하기]` | 하늘 카메라 즉시 구동 → 별자리 수집 |
| `[닫기]` | 이번 알림만 닫음 (다음 목적지에선 다시 안내) |
| `[오늘 하루 그만보기]` | 디바이스에 **자정까지 mute 플래그** 저장 → 오늘은 팝업을 강제 호출하지 않고 **백그라운드 자율 동선만 조용히 기록** |

> "또 찍어라"라고 보채지 않는다. 한 번 "오늘은 그만"이면 자정까지 침묵하되 동선 기록은 끊기지 않아,
> **알림 공해 없이도 하루의 자취는 온전히 남는다.** 사용자의 통제감이 곧 재방문의 신뢰가 된다.

**② 명소 카드 `[자세히 보기 🎧]` — 폰을 보지 않고 걸으며 듣는 오디오 도슨트**

장소 카드 하단의 `[자세히 보기 🎧]`를 누르면 빽빽한 텍스트 뷰가 아니라 **하단 시트(Bottom Sheet)
오디오 가이드 플레이어**가 떠오른다. 한국관광공사 OpenAPI의 상세 설명(`overview`)·멀티미디어
가이드를 연동해, 명소의 유래와 이야기를 **도슨트처럼 들려준다**. 멀티미디어 오디오가 없으면
온디바이스 TTS(`AVSpeechSynthesizer`, ko-KR)로 낭독해 **추가 비용 없이** 같은 경험을 제공한다.

> 화면을 응시할 필요가 없으니 걷는 힐링이 끊기지 않는다. **'읽는 관광'을 '듣는 산책'으로** 바꿔
> 명소에서의 체류 시간을 자연스럽게 늘린다.

> 💡 **심사 포인트:** 들어오는 기능(오디오 도슨트)과 **빼주는 절제(알림 그만보기)**를 한 쌍으로 설계했다.
> 사용자를 붙잡는 힘은 더 많은 알림이 아니라, **통제감과 몰입**에서 나온다.

### 2.6 K-Means 기반 감정 자동 태깅 (구현 완료)

영상 첫 프레임의 **상단 절반(하늘 영역)**만 샘플링해 scikit-learn `KMeans`로 대표색을 추출하고,
그 Hex를 **HSV 색공간으로 변환·분류**해 감정 라벨을 자동 부여한다. 사용자가 색을 고르는 일은 없다.

| 색상 영역 (HSV) | 자동 감정 라벨 |
|---|---|
| 명도 낮음(어두움) | 고요한 밤 |
| 채도 낮음(무채색) | 잔잔한 구름 |
| Hue 200–250 · 밝음 | **맑은 오후 (`#5794E4`)** |
| Hue 200–250 · 어두움 | 깊은 쪽빛 |
| Hue 250–330 (남보라) | 차분한 새벽 |
| Hue 330+ (분홍·마젠타) | 물든 노을빛 |
| Hue < 45 (붉은·주황) | 따뜻한 노을 |
| Hue 45–170 (노랑·초록) | 싱그러운 한낮 / 청량한 녹음 |
| Hue 170–200 (청록) | 청량한 물빛 |

> 이 라벨/색상은 백엔드가 생성해 응답으로 돌려주고, iOS는 이를 받아 **프로필 아바타의 별빛 색상**과
> **시간대별 하늘 그러데이션 배경**에 그대로 매핑한다. 즉 "콘텐츠의 감정"과 "UI의 색"이 한 몸으로 움직인다.

## 3. 기술적 차별성 — 가점 폭격 포인트

> 본 항목은 모두 **추상적 아이디어가 아니라 main 브랜치에 안착한 실제 동작 코드**다.
> 심사위원은 제안서가 곧 라이브 시스템임을 GitHub에서 직접 검증할 수 있다.

### 3.1 대한민국 영토 바운딩 박스 좌표 검역 시스템 — `validation.py`

GPS 노이즈·스푸핑·위변조 좌표가 동선/거리 계산을 오염시키거나 서버를 500 에러로
무너뜨리지 못하도록, **모든 좌표 입력을 영토 경계로 1차 검역**한다.

```python
# backend/app/core/validation.py
KOREA_LAT_MIN = 32.5   # 마라도 이남 여유
KOREA_LAT_MAX = 39.0   # 최북단 여유
KOREA_LNG_MIN = 124.0  # 백령도 이서 여유
KOREA_LNG_MAX = 132.5  # 독도 이동 여유

def is_in_korea(lat, lng) -> bool:
    return (is_finite_coord(lat) and is_finite_coord(lng)
            and KOREA_LAT_MIN <= lat <= KOREA_LAT_MAX
            and KOREA_LNG_MIN <= lng <= KOREA_LNG_MAX)
```

- **NaN/Inf 차단**: `math.isfinite`로 비정상 부동소수점을 1차 거부.
- **영토 밖 좌표**: `±90/±180` 범위는 통과해도 한반도 박스(마라도·독도·백령도 포함)를 벗어나면
  `400 OUT_OF_SERVICE_AREA`로 명확히 반려 — 동선 배열 중 **몇 번째 점이 문제인지**까지 알려준다.
- **파일 매직 넘버 검역**: 업로드 영상은 ISO BMFF `ftyp` 박스를 직접 읽어 실제 MP4/MOV인지 검증하고,
  HEIF/AVIF 이미지 위장 파일을 분리 거부(`ensure_video_magic`). 확장자/Content-Type만 믿지 않는다.
- **전역 예외 방어**: `debug=False` 고정 + catch-all 핸들러로 **스택 트레이스·내부 구조가
  클라이언트로 절대 새어나가지 않으며**, 일관된 `{"detail":{code,message}}` 에러 봉투만 응답한다.

> 💡 **심사 포인트:** "악의적 입력에도 서버가 죽지 않는다"는 것은 데모용 프로토타입과
> **실서비스 수준 시스템**을 가르는 결정적 기준이다. STARDUST는 이미 후자다.

### 3.2 하이브리드 위치 탐색 — PostGIS 공간쿼리 + 동적 지역 필터

§2의 듀얼 탭 탐색은 화면 장식이 아니라 **PostGIS 공간쿼리 위에 올라간 실제 API 3종**이다.
거리 계산·지역 그룹핑·통합 검색이 모두 DB 단에서 수행되어, 클라이언트는 결과만 그린다.

```
[기준 좌표]  ──┬──▶ GET /tour/spots   거리순 주변 명소 (지도 마커)
              │      ST_DWithin(geog, point, radius) + ST_Distance 정렬
              │
              ├──▶ GET /tour/search  키워드·지역 통합 검색 (리스트)
              │      ILIKE(spot_name|address) + (시/도·시/군/구) WHERE + 거리 정렬
              │
              └──▶ GET /tour/regions  시/도 → 시/군/구 동적 필터 트리
                     DISTINCT region → province/city 그룹핑
```

- **`/tour/spots` (지도)**: 기준 좌표 반경 내 명소를 `ST_DWithin`으로 추리고 `ST_Distance`로
  거리(m)를 계산해 가까운 순으로 마커를 표출. 별 마커 터치 → 장소 카드 팝업.
- **`/tour/search` (리스트)**: `keyword`는 명소명·주소 `ILIKE`, `province`/`city`는 동적 `WHERE`로
  결합. 기준 좌표가 있으면 거리순, 없으면 명소명순 정렬 + `total`/`offset` 페이지네이션.
- **`/tour/regions` (지역 필터)**: 적재된 명소의 `region`(예: "강원특별자치도 강릉시")을 `DISTINCT`로
  모아 **시/도 → 시/군/구 2단 트리**로 그룹핑 — 하드코딩 없이 데이터가 필터를 만든다.

> 💡 **심사 포인트:** "지도·리스트·검색·필터"라는 대중적 탐색 경험 전체가 **PostGIS 공간 인덱스와
> 단일 데이터 스키마 위에서 일관되게 동작**한다. UX의 편의성과 백엔드의 정합성을 동시에 증명한다.

### 3.3 강원도 우선 유도 핫트렌드 피드 — `_GANGWON_BOOST`

공모전의 핵심 정책 목표인 **지역(강원도) 관광 활성화**를 피드 정렬 알고리즘에 직접 내장했다.

```python
# backend/app/api/v1/galaxy_community.py
_GANGWON_BOOST = 50           # 강원(area_code=32) 가산점

# 최종 점수 = 실시간 동접자 수 + (강원도면 +50)
ORDER BY (
    COALESCE(a.live_count, 0)
    + CASE WHEN ts.area_code = '32' THEN 50 ELSE 0 END
) DESC, sv.created_at DESC
```

- **동접(live_users_count) 기반 실시간성**에 **강원도 가중치 +50점**을 더해,
  같은 인기도라도 강원 권역 콘텐츠가 피드 상단에 우선 노출된다.
- 단순 노출이 아니라 **"지금 사람들이 함께 보고 있는 강원의 하늘"**을 상단에 띄워
  자연스러운 방문 욕구와 후속 관광 동선을 유도한다.
- 가중치는 상수 한 줄(`_GANGWON_BOOST`)로 분리되어, 향후 캠페인·계절별 권역 프로모션에
  **코드 수정 없이 정책적으로 튜닝** 가능하다.

> 💡 **심사 포인트:** "강원도를 활성화하겠다"는 구호가 아니라, **SQL 정렬 점수 안에 정책이
> 수치로 박혀 있는** 실행 가능한 설계다.

### 3.4 신뢰·안전(Trust & Safety) — 5겹 콘텐츠 모더레이션 파이프라인

사용자가 영상을 올리는 서비스(UGC)는 **하늘이 아닌 부적절한 콘텐츠**가 유입될 위험을 반드시
설계로 막아야 한다. 이는 선택이 아니라 **App Store 심사 가이드라인 1.2의 출시 필수 요건**이다.
STARDUST는 "하늘만 올라온다"를 보장하기 위해 **비용이 낮은 단계부터 차례로 거르는 5겹 그물**을 둔다.

| 단계 | 위치 | 역할 | 상태 |
|---|---|---|---|
| **L0. 파일 정체 검역** | 서버 | 매직 넘버(ISO BMFF `ftyp`)로 실제 영상 여부 검증, 확장자 위장 거부 | ✅ **구현 완료** |
| **L1. 하늘 점수 게이트** | 서버 | 첫 프레임의 파란하늘·밝기 비율(`sky_score`)이 임계 미만이면 업로드 반려 | ◐ 계산 로직 구현(게이트 연결 예정) |
| **L2. 온디바이스 비전** | iOS | Apple `Vision`(`VNClassifyImageRequest`)으로 업로드 전 "하늘 맞음" 1차 판별 — 네트워크·비용 0 | 설계 완료 |
| **L3. 서버 NSFW 검사** | 서버 | 음란·폭력 콘텐츠 AI 판별(Google Vision SafeSearch 등 외부 API) | 설계 완료 |
| **L4. 신고·차단** | 앱+서버 | 부적절 게시물 신고, 사용자 차단 — **심사 필수 요건** | 설계 완료 |
| **L5. 속도 제한** | 서버 | 도배·스팸 업로드 차단(rate limit) | 설계 완료 |

- **이미 작동하는 1차 방어**: L0 매직 넘버 검역은 main 브랜치에서 동작 중이며(`ensure_video_magic`),
  L1의 `sky_score`(파란하늘·밝기 비율)는 색 추출 단계에서 **이미 계산**되고 있어 게이트로 즉시 승격 가능하다.
- **비용 최적화 순서**: 단말(L2)에서 거르고 → 서버 경량 휴리스틱(L0·L1)으로 거르고 → 외부 AI(L3)는
  통과분에만 호출한다. 무료에 가까운 단계가 앞단을 막아 **외부 API 비용을 최소화**한다.
- **사후 안전망**: 자동 필터를 빠져나간 콘텐츠는 사용자 신고·차단(L4)으로 회수하고, 도배는 속도 제한(L5)으로 끊는다.

> 💡 **심사 포인트:** "운영 중 유입될 악성 콘텐츠"라는 실서비스의 가장 현실적인 리스크를,
> **단계별 비용·정확도까지 고려한 파이프라인**으로 설계했다. 출시 가능성(심사 통과)을 증명하는 항목이다.

### 3.5 흉내 내지 않는 길 안내 — '감각적 최소 안내' + 국산 지도 핸드오프

목적지는 차로 가는 관광지가 아니라 **걸어서 3~5분, 100~300m 거리의 하늘 좋은 지점**이다.
그래서 거창한 턴바이턴 내비게이션을 **일부러 만들지 않았다.** 대신 둘로 나눈다.

- **앱 안 = 감각적 최소 산책 안내**: 방향 화살표 + 남은 거리 + **다음 한 구간**(MapKit `MKRoute.steps`)만
  보여준다. "지도를 봐도, 꺼두고 걸어도 좋은" 가벼운 산책 경험을 깨지 않는다.
- **정밀 길찾기 = 익숙한 국산 앱에 위임**: 길이 헷갈리면 버튼 한 번으로 **네이버지도·카카오맵·티맵**의
  도보 안내로 핸드오프한다(딥링크). 한국 사용자가 가장 신뢰하는 도구를 어설프게 흉내 내 지는 대신,
  **잘하는 건 우리가, 길찾기는 익숙한 앱이** 하게 한다.
- **도착은 GPS가 자동 감지**: `CLCircularRegion` 지오펜스로 반경 40m 진입 시 **도착 알림을 자동 발송**해,
  사용자가 "다 왔나?"를 확인할 필요가 없다(여기서 위치·알림 권한이 비로소 가치를 낸다). 단, 알림은
  `[촬영하기]·[닫기]·[오늘 하루 그만보기]` 3버튼으로 제어되어 **피로 없이 작동**한다(§2.5).

> 💡 **심사 포인트:** 기능을 "더 넣는" 결정만큼이나 **"안 만드는" 결정**도 설계다.
> 핵심 경험(별을 향한 산책)에 집중하고 주변부는 신뢰받는 생태계에 위임한 **사용자 편의 우선** 판단이다.

## 4. 비주얼 링킹 — 한반도를 품은 7개의 별자리

STARDUST의 UI는 앱 아이콘 콘셉트(파스텔 새벽/노을 오브)와 한 호흡으로 설계됐다.
**카메라 포커스 링** 안에서, 대한민국 본토와 주요 섬을 은유하는 **7개의 별이 숨 쉬듯 반짝이며
하나의 별자리**를 이룬다.

```
            ☆ (강원 — 가장 밝게 빛나는, 서비스의 주무대)
           ╱
      ☆──☆        ← 7개의 별 = 한반도 + 섬(제주·울릉·독도 등)
     ╱    ╲          포커스 링 안에서 별빛 펄스(breathe) 애니메이션
   ☆       ☆
    ╲     ╱
     ☆──☆
   ◜ 카메라 포커스 링 ◝   ← 셔터를 누르는 순간 별자리가 응답하듯 반짝임
```

### 4.1 하루의 모든 하늘빛 — 시간대 그러데이션 (SwiftUI 구현 완료)

`SkyMood` 팔레트가 K-Means 감정 라벨 11종을 **시간대 그러데이션 배경**으로 변환한다.
사용자가 올린 하늘색이 곧 앱의 분위기가 된다.

| 시간/감정 | 그러데이션 무드 | 별빛 표현 |
|---|---|---|
| 차분한 새벽 | 라벤더 → 옅은 블루 | 밝은 별이 또렷이 |
| 맑은 오후 | 청량한 스카이블루 | 햇살에 흐릿하게 |
| 따뜻한 노을 | 피치 → 로즈핑크 | 첫 별이 돋아남 |
| 고요한 밤 | 딥블루 → 인디고 | 별빛 가장 선명 |

- **`TimelineView(.animation)` + `RadialGradient` 드리프트**: 하늘빛이 미세하게 흐르며 살아 있는 배경.
- **`Canvas` 기반 60개 별 반짝임**: 각 별이 고유 위상(phase)으로 트윙클 — 정적인 이미지가 아닌 살아 있는 밤하늘.
- **`StarGlowAvatar`**: 숨쉬는 펄스(breathe) + 회전하는 오로라 링(`AngularGradient`) + 유리질 하이라이트로
  내가 올린 하늘색이 아바타 별빛으로 번지는 시그니처 인터랙션.

### 4.2 셋로그(setlog) 스타일 뷰포트 자동재생

피드를 스크롤하면 **화면 중앙에 온 셀만 무음 자동재생**되고, 벗어나는 순간 즉시
AVPlayer 인스턴스를 해제해 메모리 누수를 막는다(`AVQueuePlayer` + `AVPlayerLooper`).
사용자는 재생 버튼을 누를 필요가 없다 — 무대 위 별들이 알아서 빛난다.
